# Data Gap Handler Module v1

本模組定義資料缺口的分類、記錄格式與處置邏輯。

---

## 1. 阻斷性分類定義

| 分類 | 標記 | 定義 | 典型項目 | 處置 |
|------|------|------|---------|------|
| **Blocking** | 🛑 | 找不到就必須停在此階段，不得產出臨床建議/排除條件定稿/試驗設計依據 | 禁忌、重大警語、嚴重 ADR、DDI（尤其系統性用藥）、關鍵劑量資訊、孕哺與特殊族群禁用 | Hold |
| **Non-blocking** | ⚠️ | 可在「限制條款 + 保守假設 + 觸發更新條件」下，先做研究問題或有限度推進 | 某些次要監測頻率、少量機轉細節、次要參考文獻 | 保守假設 |

---

## 2. Blocking Data Gap 判定準則

以下項目若缺失，**必須**標記為 Blocking：

### 藥物層級（所有適應症適用）
- [ ] TFDA 仿單禁忌症
- [ ] TFDA 仿單重大警語
- [ ] 孕婦/哺乳用藥安全性
- [ ] 特殊族群（兒童、老人、肝腎功能不全）
- [ ] DDI 資料（尤其口服/全身性用藥）
- [ ] 嚴重不良反應（SAE）清單

### 適應症層級
- [ ] 臨床試驗結果（L1-L3 證據要求時）
- [ ] 目標族群定義
- [ ] 主要療效指標

---

## 3. Data Gap 完整記錄模板

每個 Data Gap 必須使用以下格式記錄：

```markdown
### Data Gap #{ID} — {缺口項目}（🛑 Blocking / ⚠️ Non-blocking）

| 欄位 | 內容 |
|------|------|
| **缺口項目** | {例：TFDA 仿單警語/禁忌} |
| **已完成檢索行動** | |
| | • 官方標籤/主管機關資料（TFDA/FDA/EMA/PMDA）：{結果} |
| | • 權威藥學資料庫（DDInter/Guide to PHARMACOLOGY/DrugBank）：{結果} |
| | • 同儕審查文獻（PubMed/期刊）：{結果} |
| | • 試驗登錄（ClinicalTrials.gov）：{結果} |
| **檢索結果** | 未找到 / 僅二手資訊 / 資訊互相矛盾 |
| **推論限制（不能做）** | {明確列出禁止輸出的結論} |
| **風險處置（決策）** | Hold / Research Question only / Proceed with Guardrails |
| **解鎖條件** | {需要哪種資料、補到哪個欄位後才可升級階段} |
```

---

## 4. 保守假設格式（Non-blocking 專用）

當缺口為 Non-blocking 且需往下推進時，必須標示：

```markdown
### 保守假設：{假設項目}

| 欄位 | 內容 |
|------|------|
| **假設（Assumption）** | 我們暫時假設 X |
| **依據（Rationale）** | 基於同類藥/同機轉/同途徑的已知風險 |
| **限制（Limitation）** | 若假設不成立，可能造成的偏差方向（高估/低估風險） |
| **觸發條件（Trigger）** | 一旦找到相反證據，就必須更新哪些段落與結論 |
```

---

## 5. 三種決策結論

取代原本的「待補」，必須落到以下三種之一：

### A. Hold（有 Blocking Gap）
```markdown
### 📋 決策結論：Hold

**阻斷原因**：存在 {N} 個 Blocking Data Gap
**補件清單**：
1. [ ] {Gap 1} — 來源：{來源}
2. [ ] {Gap 2} — 來源：{來源}

**重新評估時機**：補齊上述資料後
```

### B. Research Question（證據不足）
```markdown
### 🔬 決策結論：Research Question

**定位**：本案例應作為研究問題，不適合直接臨床應用
**待回答問題**：{Q1, Q2...}
**建議研究設計**：{設計類型}
```

### C. Proceed with Guardrails（有護欄）
```markdown
### ✅ 決策結論：Proceed with Guardrails

**護欄條件**：詳見【Guardrails 章節】
- 適用範圍：{定義}
- 排除條件：{定義}
- 監測 Stop Rule：{定義}
- 告知義務：{定義}
```

---

## 6. 資料缺口三分類標記

| 狀態 | 標記 | 定義 |
|------|------|------|
| 未檢索 | **[未檢索]** | 本輪資料收集未查詢該來源 |
| 未找到 | **[未找到]** | 已查詢但未找到相關資料 |
| 不適用 | **[不適用]** | 該項目不適用於此情境 |

**絕對禁止**：「無」、「無特別」、「無風險」

---

## 7. 決策階段與 Data Gap 的關係

| 決策階段 | 定義 | 必須無 Blocking Gap 的項目 |
|---------|------|--------------------------|
| S0 初篩 | 模型預測 + 基本資料 | - |
| S1 安全性初評 | 可接受安全性風險 | 禁忌、重大警語、DDI |
| S2 臨床可行性評估 | 有機轉支持 + 初步證據 | S1 + 臨床試驗/文獻 |
| S3 用藥/開發建議 | 完整評估 | S2 + 監測計畫 + 告知 |

---

## 8. 跨法規暫代機制

**適用情境**：TFDA 資料缺失時

**處理步驟**：
1. 先確認：是否因為品項/劑型/濃度/學名藥不同導致找不到
2. 以 FDA/EMA/PMDA 任一官方 label 暫代
3. 必須標示差異與限制

```markdown
### ⚠️ 跨法規暫代資料

**缺失項目**：TFDA {項目}
**暫代來源**：{FDA DailyMed / EMA SmPC / PMDA}
**暫代內容摘要**：{摘要}

**版本稽核**：
| 項目 | 台灣預期 | 暫代來源 | 差異說明 |
|-----|---------|---------|---------|
| 劑型 | {台灣劑型} | {來源劑型} | {差異} |
| 濃度 | {台灣濃度} | {來源濃度} | {差異} |
| 適應症 | {台灣適應症} | {來源適應症} | {差異} |
| 警語 | [缺失] | {來源警語} | {適用性說明} |

**暫代限制**：
- ⚠️ 此為非台灣法規資料
- ⚠️ 與台灣預期使用情境可能不同之處已標示
- ⚠️ 當 TFDA 資料可得時**必須更新**

**觸發更新條件**：取得 TFDA 官方仿單後，需重新比對並更新以下章節：{列出章節}
```

**優先順序**：TFDA → FDA → EMA → PMDA
