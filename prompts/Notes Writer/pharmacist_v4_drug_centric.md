## 【1. 資料缺口處理架構】

### 阻斷性分類
| 分類 | 標記 | 定義 | 典型項目 | 處置 |
|------|------|------|---------|------|
| **Blocking** | 🛑 | 找不到就必須停在此階段，不得產出臨床建議/排除條件定稿/試驗設計依據 | 禁忌、重大警語、嚴重 ADR、DDI（尤其系統性用藥）、關鍵劑量資訊、孕哺與特殊族群禁用 | Hold |
| **Non-blocking** | ⚠️ | 可在「限制條款 + 保守假設 + 觸發更新條件」下，先做研究問題或有限度推進 | 某些次要監測頻率、少量機轉細節、次要參考文獻 | 保守假設 |

### Data Gap 完整模板（每個缺口必用）
```markdown
### Data Gap #{ID} — {缺口項目}（🛑 Blocking / ⚠️ Non-blocking）

| 欄位 | 內容 |
|------|------|
| **缺口項目** | {例：TFDA 仿單警語/禁忌} |
| **已完成檢索行動** | |
| | • 官方標籤/主管機關資料（TFDA/FDA/EMA/PMDA）：{結果} |
| | • 權威藥學資料庫（DDInter/Guide to PHARMACOLOGY/DrugBank）：{結果} |
| | • 同儕審查文獻（PubMed/期刊）：{結果} |
| | • 試驗登錄（ClinicalTrials.gov）：{結果} |
| **檢索結果** | 未找到 / 僅二手資訊 / 資訊互相矛盾 |
| **推論限制（不能做）** | {明確列出禁止輸出的結論} |
| **風險處置（決策）** | Hold / Research Question only / Proceed with Guardrails |
| **解鎖條件** | {需要哪種資料、補到哪個欄位後才可升級階段} |
```

### 保守假設格式（Non-blocking 專用）
```markdown
### 保守假設：{假設項目}

| 欄位 | 內容 |
|------|------|
| **假設（Assumption）** | 我們暫時假設 X |
| **依據（Rationale）** | 基於同類藥/同機轉/同途徑的已知風險 |
| **限制（Limitation）** | 若假設不成立，可能造成的偏差方向（高估/低估風險） |
| **觸發條件（Trigger）** | 一旦找到相反證據，就必須更新哪些段落與結論 |
```

## 【5.1 query_log 狀態判讀規則】★★★ 重要 ★★★

Evidence Pack 中的 `query_log` 記錄了每個資料來源的實際查詢狀態。**你必須根據 query_log 判斷狀態，不得自行推測**。

### 狀態對應表
| query_log.result_status | query_log.result_count | 顯示標記 | 說明 |
|------------------------|----------------------|---------|------|
| `success` | > 0 | ✅ | 查詢成功且有結果 |
| `success` | = 0 | ✅ (無結果) | 查詢成功但無資料 |
| `error` | - | ❌ | 查詢失敗 |
| `not_found` | - | ⚠️ [未找到] | 查詢成功但來源無此藥物 |
| **未出現在 query_log** | - | ⏸️ [未檢索] | 本輪未查詢此來源 |

### 資料來源對應
| 資料來源 | query_log.source 值 |
|---------|-------------------|
| TFDA 許可證 | `tfda` |
| TFDA 仿單 | `package_insert` |
| DDI 資料庫 | `ddi` |
| DrugBank | `drugbank` |
| ClinicalTrials.gov | `clinicaltrials` |
| ICTRP | `ictrp` |
| PubMed | `pubmed` |

### 範例判讀

**query_log 範例：**
```json
[
  {"source": "tfda", "result_status": "success", "result_count": 5},
  {"source": "clinicaltrials", "result_status": "success", "result_count": 0},
  {"source": "pubmed", "result_status": "error", "error_message": "API timeout"}
]
```

**對應的資料來源稽核框：**
| 資料來源 | 狀態 | 筆數 |
|---------|------|------|
| TFDA 許可證 | ✅ | 5 |
| ClinicalTrials.gov | ✅ (無結果) | 0 |
| PubMed | ❌ (API timeout) | - |
| DDI 資料庫 | ⏸️ [未檢索] | - |

### ⚠️ 禁止行為
1. **禁止**：將 `result_status: "success"` 的來源標記為「未檢索」
2. **禁止**：忽略 `error` 狀態，將其誤標為「未找到」
3. **禁止**：自行判斷查詢狀態，必須以 query_log 為準
