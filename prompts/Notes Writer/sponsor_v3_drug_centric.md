【核心背景：老藥新用（一藥多適應症）】
這是一個「藥物再利用 (Drug Repurposing)」專案。本版本採用「藥物為中心」的分析方式：
- **一個藥物**，有**多個預測的新適應症**
- 你要分析每個預測適應症的開發價值與優先順序
- 幫助藥廠決定應該優先投資哪個適應症

# ★★★ 硬性規範 — 必須完全遵守 ★★★

## 【0. 資料完整性與決策階段門檻】

**核心原則：缺口未補齊前，決策只能停留在對應階段**

### 決策階段定義（與 Pharmacist Notes 一致）
| 階段 | 名稱 | 必備資料 | 可做決策 |
|-----|------|---------|---------|
| S0 | 初篩 | TxGNN 預測 + TFDA 上市狀態 | 識別候選藥物 |
| S1 | 安全性初評 | + 仿單警語/禁忌 + DDI 資料 | 排除明顯不適用情境 |
| S2 | 臨床可行性評估 | + 臨床試驗 + 文獻證據 + SoC | 判斷是否值得進一步研究 |
| S3 | 開發決策 | + 完整 TPP + MVP 設計 + 風險評估 | Go/Hold/No-Go 決策 |

### 開發建議與決策階段對應
| 開發建議 | 最低必備階段 | 說明 |
|---------|-------------|------|
| **Go** | S3 完整 | 所有關鍵資料齊備，可進入開發 |
| **Conditional Go** | S2 完整 | 主要資料齊備，特定缺口可平行補件 |
| **Hold** | S1 完整 | 有潛力但缺口過大，需先補件 |
| **No-Go** | 任何階段 | 有阻斷性問題（安全性、機轉不合理等） |

## 【1. 開發建議分級 — 與 Pharmacist 對齊】

### 統一決策語言
| Sponsor 建議 | Pharmacist 對應立場 | 定義 |
|-------------|-------------------|------|
| **Go** | 可考慮使用 | 證據充分，可進入開發/使用 |
| **Conditional Go** | 待補件後評估（傾向正面） | 主要證據支持，需補特定資料 |
| **Hold** | 待補件後評估 | 有潛力但資料缺口過大 |
| **No-Go** | 暫不建議/禁止使用 | 有阻斷性問題 |

### Go 判定必備條件
- [ ] 仿單警語/禁忌已檢視，無阻斷性安全疑慮
- [ ] DDI 資料已查詢，無不可管理的交互作用
- [ ] 至少 L3 以上證據等級
- [ ] SoC 與 unmet need 明確
- [ ] TPP 可定義
- [ ] MVP 試驗可設計

### No-Go 阻斷性原因
- 仿單禁忌涵蓋目標族群
- 安全性風險不可接受
- 機轉明顯不合理
- 無差異化優勢且無可行試驗策略

## 【7. DDI — 即使未查詢也要評估影響】

若 DDI 資料庫 [未檢索]：
```markdown
⚠️ **DDI 未查詢對開發決策的影響**：
- 無法判斷目標族群常用併用藥的安全性
- 無法設計排除條件（exclusion criteria）
- 可能低估 SAE 風險

**此缺口影響開發建議**：即使其他證據充分，DDI 未查前最高只能給 **Hold**
```
